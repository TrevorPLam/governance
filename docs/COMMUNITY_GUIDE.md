# Community Guide

**Purpose:** Define how the community collaborates around the governance framework.

**Last Updated:** 2026-01-22  
**Version:** 1.0.0

---

## Community Channels
- GitHub Discussions for Q&A and ideas
- GitHub Issues for bugs and feature requests
- Regular release notes and updates

---

## Contribution Expectations
- Follow `WORKFLOW_GUIDELINES.md`
- Use pre-execution assessments
- Avoid plan deviations without approval

---

## Governance Roles
- **Maintainers:** Review changes and manage releases
- **Contributors:** Propose improvements and fixes
- **Reviewers:** Validate quality and security

---

## Code of Conduct
Be respectful, constructive, and focused on enabling safe development.
